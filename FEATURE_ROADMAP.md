## 当前代码中的重要实现约束

### 全量索引是前端驱动的

当前没有 `/index/full`。全量索引由插件：

1. 枚举 Markdown 文件
2. 本地清洗文本
3. 每批 50 篇调用 `/index/batch`

因此：

- 进度是插件内状态，不是后端持久任务
- 关闭 Obsidian 或禁用插件后，进度不会保留

### 搜索目前是纯向量检索

当前检索链路是：

1. 查询文本加 BGE 指令前缀
2. LanceDB cosine search
3. 根据 path 去重并聚合为文档级结果

这意味着：

- 对语义相关内容效果较好
- 对专有名词、缩写、代码符号的召回仍可能不足

### 排除规则只是路径前缀匹配

设置页里的 `Exclusion Rules` 当前不是完整 glob 语法，也不是正则。它的真实行为是：

- 每行一条规则
- 只判断 `path.startsWith(rule)`

文档与 UI 后续都应继续保持这一点的明确说明，直到实现升级。

## 下一阶段路线

### P0

- 混合检索
  - 向量检索补上关键词召回
  - 重点改善专有名词与短查询命中
- 排除规则升级
  - 明确支持 glob 或正则中的一种
  - 同步更新 UI 文案和迁移策略

### P1

- cross-encoder rerank
  - 先粗召回，再精排
  - 提升 Top 5 结果排序稳定性
- 孤岛笔记专用推荐策略
  - 引入标题、标签、局部上下文等额外特征

### P2

- 知识图谱可视化
  - 基于现有链接与推荐关系展示关联网络
- 更细的运维与诊断能力
  - 索引耗时拆分
  - 文档数 / chunk 数监控
  - 失败批次定位
