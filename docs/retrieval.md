# 检索与召回逻辑详解

Semantix 的核心竞争力在于其针对知识库场景优化的两阶段检索与聚合策略。

## 1. 文本预处理 (Preprocessing)

为了保留 Markdown 的结构语义同时移除干扰噪声，前端在同步前会执行轻量化清洗：
- **保留项**：Markdown 标题 (#)、换行符、列表标记、链接文字、双链别名。
- **移除项**：YAML 元数据、媒体嵌入 (![[]])、HTML 标签、加粗/斜体等装饰符。
- **保护项**：代码块内容会被保留，仅移除围栏标记。

## 2. 标题感知切块 (Header-aware Chunking)

后端不直接对整文件进行编码，而是执行智能切块：
- **Parent Chunk**：按物理段落（空行）拆分，作为最终展示 snippet 的上下文。
- **Child Chunk**：为提升粒度，对长段落进一步切分。
- **语义注入**：每个块在编码前都会自动注入「文件名 + 最近层级标题」作为前缀。
  - *示例*：`[Python 基础] [列表操作] ...实际内容`

## 3. 混合检索策略 (Hybrid Search)

系统根据查询文本长度动态切换算法：

| 模式 | 触发条件 | 效果 |
| --- | --- | --- |
| **Hybrid** | 查询 ≤ 128 字符 | 同时发起向量检索与关键词检索 (FTS)，利用 LinearCombinationReranker 融合。 |
| **Pure Vector** | 查询 > 128 字符 | 仅使用向量检索，避免长文本在 FTS 中引起的过度噪声和延迟。 |

## 4. 文档级聚合评分 (Aggregation)

检索是以块 (Chunk) 为单位的，但返回结果以文档 (File) 为单位：
1. **去重聚合**：同一文件的多个命中块会合并。
2. **基础分**：取该文件下评分最高的一个块作为 Base Score。
3. **命中增益 (Hit-boost)**：每额外命中一个不同位置的块，得分增加 0.02 (上限 0.06)。
   - *逻辑*：一篇文档中有三个段落同时与查询相关，其排名应显著高于仅有一个相关段落的文档。

## 5. 结果解释与展示
后端返回最匹配块的局部文本作为 Snippet。前端根据最终分值映射颜色指示灯：
- 🟢 >= 0.85 (强相关)
- 🔵 >= 0.75 (相关)
- 🟡 < 0.75 (潜在关联)
