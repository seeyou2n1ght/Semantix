## [0.4.3] - 2026-04-10

### 🚀 新功能 (Features)
- **动态状态追踪 (Live Status Tracking)**: 引入基于单条动态 Notice 的进度追踪系统。在后端启动全周期（同步依赖、唤醒服务、加载模型）提供原生的右上角实时反馈。
- **智能日志解析**: 升级 `ServiceManager` 能够实时流式解析 Python 进程的 stdout/stderr，捕捉耗时较长的“哑火期”（如模型加载）并及时播报。

### 🩹 修复 (Fixes)
- 修正了 `ServiceManager` 类定义中因多次逻辑注入导致的语法结构错误。
- 优化了提示文本的语义，使“手动启动”与“探测连接”的反馈更加符合直觉。

## [0.4.6] - 2026-04-10

### 🚀 优化 (Optimizations)
- **浮窗生命周期管理**: 引入状态锁机制，彻底解决启动浮窗在服务就绪后因后续杂散日志而“复活”并驻留的问题。
- **心跳探测联动**: 实现 `checkConnection` 成功后强制清理启动 Notice 的逻辑，确保 UI 状态（已连接）与右上角反馈严格同步。
- **高精度日志解析**: 
    - 增加对模型下载进度（`Downloading: XX%`）的实时捕获。
    - 细化了 `uv sync` 的环境同步阶段（解析、准备、安装）反馈。

## [0.4.5] - 2026-04-10

### 🚀 优化 (Optimizations)
- **状态实时同步 (Real-time Status Sync)**: 增强了设置面板与插件核心的状态订阅机制。设置面板现在能自动响应后台心跳探测，实时展示连接状态指示灯，无需手动刷新。
- **UI 增强**: 在设置页面顶部增加了全局状态徽标，直观展示已连接、未连接、同步中等实时状态。

### 🩹 修复 (Fixes)
- **进程生命周期加固**: 
    - 将 Windows 进程清理逻辑改为同步执行 (`execSync`)，彻底解决了容器/Obsidain 退出时异步回收任务被中断导致的进程残留问题。
    - **智能特征匹配**: 引入了基于命令行指纹的进程校验。清理端口冲突时会核对路径与特征字符串，确保**绝不误杀**用户系统中其他无关的 Python 或 Web 进程。



## [0.4.4] - 2026-04-10

### 🚀 优化与修复
- **进程回收优化**: 在 Windows 平台改用进程树清理 (`taskkill /T`)，确保后端进程及其子进程在插件卸载时能完全退出。
- **启动性能优化**: 将后端启动过程改为非阻塞异步执行，避免在自动拉起时占用 Obsidian 的初始化时间。
- **平台兼容性**: 为所有底层进程操作增加平台校验，确保在移动端环境下不触发无效调用。

## [0.4.1] - 2026-04-10

### 🚀 新功能 (Features)
- **并发检索版本控制 (Search Versioning)**: 实现在逻辑层通过闭包版本号校验，丢弃过时的异步请求，彻底解决高频操作下的结果跳闪现象。
- **边车服务自启动 (Sidecar Service)**: 支持随 Obsidian 自动启动/销毁本地 Python 后端，极大降低了本地运行门槛。
- **双模式设置面板**: 引入本地/远程双模式切换，实现设置项的动态按需渲染。
- **孤岛雷达交互增强**: 引入基于 CSS Class 的悬停（Hover）高亮反馈与 `pointer` 指针状态。
- **检索上下文增强 (Metadata Enrichment)**: 在语义搜索请求中自动注入笔记标题、Tags 及文件路径，极大提升了对短文本或零散笔记的推荐准确度。

### 🛠️ 架构与工程 (Engineering)
- **文档体系重塑**: 建立 `docs/` 专项手册体系，实现技术细节（检索、架构、API）与入门指南的完全解耦。
- **样式系统重构**: 实现了 `styles.css` 构建集成，全量弃用 JS 硬编码样式，完美适配 Obsidian 原生暗色/亮色主题。
- **构建链路优化**: 升级 `esbuild` 配置支持多入口异步构建，自动压缩输出 CSS 产物。
- **CI/CD 修复**: 
    - 解决了 `npm ci` 因 `picomatch` 版本不一致（Lockfile 不同步）导致的构建异常。
    - 建立了前端构建产物（main.js, styles.css）的自动打包流程。

### 🩹 修复 (Fixes)
- **切块结构解析**: 优化了标题感知切块算法，修复了切块过程中可能导致的 Markdown 结构丢失问题。
- **Obsidian 规范对齐**: 将所有侧边栏视图标题、设置项名称统一调整为 **Sentence case**。
- **类型安全**: 全量修复了 CI 中的 Lint 报错，包括 `any` 类型限制、`AbortError` 型别校验以及 `console` 日志规范。
