# Semantix 演进路线与当前状态

本文档记录 Semantix 的系统现状、已知约束以及未来的开发计划。

## 1. 当前实现 (Current Status)

### 已实现功能 (Implemented)

- **核心检索**
  - [x] 基于示例的 `Whisperer` 实时语义推荐
  - [x] `Orphan Radar` 孤岛笔记扫描与分析
  - [x] 基于 LanceDB FTS 的混合检索 (Hybrid Search)
  - [x] 索引重建节流与 FTS 索引状态管理
  - [x] 文档级聚合与多片段命中 Boost 算法
- **同步与管理**
  - [x] Vault 级哈希隔离
  - [x] 全自动增量同步 (create/modify/rename/delete)
  - [x] 全量异步初始化索引 (前端驱动)
  - [x] 双步确认的安全清空索引机制
- **细节优化**
  - [x] 标题感知 (Header-aware) 的 Markdown 切块算法
  - [x] BGE 查询指令自动前缀注入
  - [x] 可解释的结果：Snippets 提取与关键词高亮
  - [x] 移动端自动化休眠保护
  - [x] Markdown 代码块语义保留处理
  - [x] 后端优雅停机 (Graceful Shutdown) 保护数据库
  - [x] Cross-encoder Rerank 精排重排 (BGE-Reranker)
  - [x] 智能关键词高亮 (Intl.Segmenter + 权威停用词库)
  - [x] 启发式噪音过滤 (基于文档频率的自适应停用词)

## 2. 计划中功能 (Roadmap)

### 近期计划 (P1)
- [ ] **性能观测**：建立固定查询样本集，记录各阶段 (encode/search/rerank) 延迟日志。
- [ ] **结构化 Boost**：为 `title` / `tags` / `headers` 提供独立权重的结构化评分。
- [ ] **移动端增强**：优化移动端增量同步的功耗。

### 中期计划 (P2)
- [ ] **图谱集成**：实现基于语义关联核心的知识图谱可视化。
- [ ] **异步任务系统**：将全量索引重构为后端驱动的长生命周期异步任务。

---

## 3. 当前约束 (Current Constraints)

- **前端驱动的索引**：目前全量索引依赖插件前端循环触发，意味着进度存在于内存中，关闭 Obsidian 会重置进度。
- **本地性能依赖**：模型推理与向量计算均在本地完成，首次加载或超大规模库 (10k+) 索引时 CPU 开销较大。

---

## 4. 检索优化专项

我们计划分阶段提升检索的响应速度与相关性。

- **Phase 1: 结构保留**：在前置清洗中保留标题与段落边界，确保 Header-aware 切块有效。 (已完成)
- **Phase 2: 消除冗余**：移除无效的重复 Embedding 调用。 (已完成)
- **Phase 3: 策略收紧**：根据长度自动切换 Hybrid/Vector 模式。 (已完成)
- **Phase 4: 聚合增强**：放大候选池并优化文档级聚合评分。 (已完成)
- **Phase 5: 高亮降噪**：智能分词 + 自适应噪音过滤。 (已完成)
